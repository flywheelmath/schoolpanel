## multi-step linear

* unbracketed
  - this is recurrent
    * combine like terms
    * cancel smaller variable term
* bracketed
  - this might be non-recurrent/heuristic
    * always combine like terms when possible
    * if the coefficient of the brackets divides the other side, reciprocal
    * otherwise, distribute
    * malrule: 3 + 5(x + 2) -> 8(x + 2)
* rational
  - in theory, this should be non-recurrent/heuristic, but is too taxing on working memory
    * always combine like terms when possible
    * separate the denominators
